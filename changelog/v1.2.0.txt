# Argus v1.2.0 Release Notes

**Release Date**: February 2, 2026  
**Type**: Feature Release

## Overview

This release adds comprehensive **directory watching** capabilities, enabling Argus to scan entire directories for configuration files with pattern filtering, recursive subdirectory support, and hot-reload detection for new, modified, or deleted files.

This enables powerful use cases like:
- Policy engines loading rules from `/etc/app/policies/`
- Configuration layering with `00-base.yaml`, `10-override.yaml` patterns
- Dynamic discovery of new configuration files without restart

## New Features

### Directory Watching (`WatchDirectory`)

Watch an entire directory for configuration files with pattern-based filtering:

```go
watcher, err := argus.WatchDirectory("/etc/myapp/config.d", argus.DirectoryWatchOptions{
    Patterns:  []string{"*.yaml", "*.yml", "*.json"},
    Recursive: true,
    PollInterval: 5 * time.Second,
}, func(update argus.DirectoryConfigUpdate) {
    if update.IsDelete {
        log.Printf("Config removed: %s", update.FilePath)
    } else {
        log.Printf("Config updated: %s", update.FilePath)
        applyConfig(update.Config)
    }
})
defer watcher.Close()
```

**Key capabilities:**
- **Pattern filtering**: Support for multiple glob patterns (`*.yaml`, `*.json`, `*.toml`, etc.)
- **Recursive**: Option to watch subdirectories
- **Hot-reload**: Automatic detection of new files, modifications, and deletions
- **Parsed content**: Each update includes the parsed configuration (`map[string]interface{}`)
- **Format auto-detection**: Works with all supported formats (YAML, JSON, TOML, INI, etc.)

### Merged Directory Watching (`WatchDirectoryMerged`)

For configuration layering, merge all files in alphabetical order:

```go
watcher, err := argus.WatchDirectoryMerged("/etc/myapp/config.d", argus.DirectoryWatchOptions{
    Patterns: []string{"*.yaml"},
}, func(merged map[string]interface{}, files []string) {
    // Files merged in order: 00-base.yaml, 10-app.yaml, 20-override.yaml
    applyMergedConfig(merged)
})
```

**Use cases:**
- Configuration precedence (later files override earlier)
- Environment-specific overlays
- Plugin/module configuration discovery

### DirectoryWatchOptions

New configuration struct for directory watching:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `Patterns` | `[]string` | `["*.yaml", "*.yml", "*.json", "*.toml", "*.ini"]` | Glob patterns to match |
| `Recursive` | `bool` | `false` | Watch subdirectories |
| `PollInterval` | `time.Duration` | `1s` | Directory scan interval |
| `Context` | `context.Context` | `nil` | For cancellation |

### DirectoryConfigUpdate

Update event structure:

| Field | Type | Description |
|-------|------|-------------|
| `FilePath` | `string` | Absolute path to file |
| `RelativePath` | `string` | Path relative to watched directory |
| `Config` | `map[string]interface{}` | Parsed configuration |
| `Format` | `string` | Detected format ("yaml", "json", etc.) |
| `IsDelete` | `bool` | True if file was deleted |
| `ModTime` | `time.Time` | File modification time |

## Security

### Red Team Tested

The directory watching feature has been subjected to comprehensive security testing:

- **Path traversal attacks**: Blocked via `..` detection
- **Symlink escapes**: Files outside watched directory not accessible via symlinks
- **Resource exhaustion**: Tested with 100-level deep directories and 1000+ files
- **Race conditions (TOCTOU)**: Graceful handling of rapid file changes
- **Permission handling**: Unreadable files skipped gracefully
- **Malformed filenames**: Unicode, emojis, long names handled safely
- **Injection payloads**: Config content treated as data, never executed

### Security Tests Added

New test file `directory_watcher_security_test.go` with:

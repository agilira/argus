NEW
---

1. Config.DisableAudit — inert audit logger for hosts with their own trail

       DisableAudit bool

   When true, argus.New installs an INERT audit logger: no SQLite backend is
   opened (no system-audit.db under TempDir) and no flush goroutine is started.

   WHY: every argus.New otherwise stands up its own unified system audit logger
   (createAuditBackend opens the shared TempDir/argus/system-audit.db, and a
   FlushInterval>0 starts a flush goroutine). A host that runs several watchers
   in one process — and that already owns its own audit trail — ends up with
   several redundant system-audit databases and flush goroutines it never asked
   for. DisableAudit is the clean opt-out.

   Precedence: DisableAudit overrides Audit. The secure default is unchanged —
   leave it false (zero value) to keep audit enabled with secure settings.

       // host owns its own audit; tell argus not to open a second one
       w := argus.New(argus.Config{
           OptimizationStrategy: argus.OptimizationLight,
           DisableAudit:         true,
       })


### Technical Details

setAuditDefaults now short-circuits on DisableAudit:

    func (c *Config) setAuditDefaults() {
        if c.DisableAudit {
            c.Audit = AuditConfig{Enabled: false}
            return
        }
        if c.Audit == (AuditConfig{}) {
            c.Audit = DefaultAuditConfig()
        }
    }

argus.New routes through a new newDisabledAuditLogger() when DisableAudit is
set, instead of NewAuditLogger (which always opens a backend). The inert logger
is safe on every method: Log returns early on the nil backend, Flush sees an
empty buffer, GetStats/Query return a typed error, Close is a no-op beyond
closing stopCh.


COMPATIBILITY
-------------

Additive: a new optional Config field defaulting to false. Existing
configurations behave exactly as before — audit stays enabled by default.
